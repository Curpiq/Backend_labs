2. 
a) 
Status Code: 404 Not Found
Ошибка: Клиент был в состоянии общаться с сервером, но сервер не может найти данные согласно запросу
Исправить ситуацию можно, разместив файл по указанному адресу
b) 
Status Code: 304 Not Modified
Код клиента указывает, что нет необходимости повторно передавать запрошенные ресурсы. 
Это неявное перенаправление на кэшированный ресурс.
Специальное поведение
с)
Status Code: 302 Found
Указывает на то, что запрошенный документ временно доступен по другому URI, указанному в заголовке в поле Location. 
Специальное поведение
d)
HTTP ERROR 400
Код указывает, что сервер не смог понять запрос из-за недействительного синтаксиса. 
Исправить hardly на hard: http://localhost/lw1/javascript_is_a_client_side_language.php?how_to_work=hard

3. 
Параметр charset требуется для того, чтобы указать кодировку символов документа. 
В HTTP 1.1 говорится, что по умолчанию charset должен быть ISO-8859-1. 
В данном примере кодировка файла (UTF-8) не совпадает с параметром charset(Windows-1252). 
Для решения проблемы в файле в <head> надо прописать <meta charset="utf-8" />
